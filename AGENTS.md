# Codex project workflow

- Надавай фінальні звіти українською мовою.
- Перед глобальним завданням фіксуй повний план фаз із сумарною вагою 100%.
- Після кожної фази показуй progress bar, зміну прогресу, blockers і regression.
- Мінімізуй кількість ітерацій, токени й загальний час; сумісні роботи об’єднуй без втрати якості.
- Програмування виконуй однією фазою, максимум двома лише через об’єктивну зовнішню залежність.
- Звичайні defects виправляй у поточній фазі; не створюй окрему фазу для кожного defect або blocker.
- Не переходь до наступної фази без прямої авторизації owner.
- Не виконуй push, merge, deploy, production switch або destructive actions без прямого дозволу.
- Не використовуй force push, rebase або amend без окремої авторизації.
- Перед змінами перевіряй Git і database baseline; не змінюй normal або production data поза scope.
- Визначай acceptance criteria пропорційно реальному ризику та масштабу.
- Конкретне поточне розпорядження має пріоритет над цим файлом.
